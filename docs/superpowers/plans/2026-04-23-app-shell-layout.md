# App Shell Layout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the persistent `<header> / <aside> / <main> / <footer>` app shell for rmtp-studio as a single CSS grid driven entirely by CSS custom properties, so the existing themable-studio pipeline can white-label the shell per tenant without JSX changes.

**Architecture:** One CSS grid on a single root `<div>` fills `100dvh`, with named grid areas. Header and footer rows are fixed-height; the middle row is `minmax(0, 1fr)`. Aside and main declare `overflow: auto` so they are the only scroll containers — header/footer never move because the grid itself does not overflow the viewport. Below 768px a CSS-only `@media` rule collapses the grid and reveals a centered "Desktop only" screen.

**Tech Stack:** React 19, Tailwind v4 (`@theme` tokens generated by themable-studio), TanStack Start/Router, vitest + @testing-library/react (jsdom via per-file pragma), Biome (single-quote JS), `dangerouslySetInnerHTML`-based `<Logo />` already provided by `src/components/logo.tsx`.

**Spec:** `docs/superpowers/specs/2026-04-23-app-shell-layout-design.md`

**Pre-flight assumption:** Themable-studio work is already merged on this branch — `src/components/logo.tsx`, `src/styles/theme.generated.ts`, and `src/styles/theme.generated.css` exist and are up to date. (Verified at plan-write time on `feat/themable-studio`.)

## Task 1: Add layout tokens and shell grid rules to `src/styles.css`

**Files:**

- Modify: `src/styles.css`

- [ ] **Step 1: Add new layout tokens to the existing `html` block in `@layer base`**

Open `src/styles.css`. Find the existing `@layer base { html { --header-height: 64px; ... } }` block (around lines 8–37). Immediately after the `--layout-separator-size: 12px;` line and before the `--ease-in-quad` line, insert the shell tokens so the block reads (new lines only shown below; keep every other existing line in place):

```css
    --header-height: 64px;
    --sidebar-width: 300px;
    --content-height: calc(100vh - var(--header-height));
    --layout-separator-size: 12px;

    /* shell layout — all overridable per theme */
    --footer-height: 40px;
    --shell-padding: var(--layout-separator-size);
    --shell-gap: var(--layout-separator-size);
    --shell-bg: var(--color-background);

    --panel-bg: var(--color-gray-2);
    --panel-radius: 0.75rem;

    --unsupported-bg: var(--color-background);
    --unsupported-padding: calc(var(--layout-separator-size) * 2);
    --unsupported-max-inline-size: 32ch;
```

Leave the existing `--ease-*` vars, `interpolate-size: allow-keywords`, and the `img { ... }` rule unchanged.

- [ ] **Step 2: Add a new `@layer components` block at the end of `src/styles.css`**

Append the following to the end of the file, after the existing `@layer layout { ... }` block closes:

```css
@layer components {
  .app-shell {
    display: grid;
    block-size: 100dvh;
    inline-size: 100%;

    grid-template-columns: var(--sidebar-width) minmax(0, 1fr);
    grid-template-rows:
      var(--header-height)
      minmax(0, 1fr)
      var(--footer-height);
    grid-template-areas:
      "header header"
      "aside  main"
      "footer footer";

    gap: var(--shell-gap);
    padding: var(--shell-padding);
    background-color: var(--shell-bg);
  }

  .app-shell > .app-shell__header {
    grid-area: header;
  }
  .app-shell > .app-shell__aside {
    grid-area: aside;
  }
  .app-shell > .app-shell__main {
    grid-area: main;
  }
  .app-shell > .app-shell__footer {
    grid-area: footer;
  }

  .app-shell
    > :is(
      .app-shell__header,
      .app-shell__aside,
      .app-shell__main,
      .app-shell__footer
    ) {
    background-color: var(--panel-bg);
    border-radius: var(--panel-radius);
  }

  .app-shell__aside,
  .app-shell__main {
    overflow: auto;
    min-block-size: 0;
    min-inline-size: 0;
  }

  .app-shell__unsupported {
    display: none;
  }

  .unsupported-screen {
    display: flex;
    flex-direction: column;
    align-items: center;
    gap: var(--layout-separator-size);
    max-inline-size: var(--unsupported-max-inline-size);
    text-align: center;
    color: var(--color-gray-12);
  }
  .unsupported-screen__title {
    font-family: var(--font-display);
  }
  .unsupported-screen__body {
    color: var(--color-gray-11);
  }

  @media (max-width: 767px) {
    .app-shell {
      display: block;
      padding: 0;
      background-color: var(--unsupported-bg);
    }
    .app-shell
      > :is(
        .app-shell__header,
        .app-shell__aside,
        .app-shell__main,
        .app-shell__footer
      ) {
      display: none;
    }
    .app-shell__unsupported {
      display: grid;
      place-items: center;
      block-size: 100dvh;
      padding: var(--unsupported-padding);
    }
  }
}
```

## Task 3: Write failing test for `AppShell`

**Files:**

- Create: `src/components/__tests__/app-shell.test.tsx`

- [ ] **Step 1: Write the failing test**

Path: `src/components/__tests__/app-shell.test.tsx`

The `// @vitest-environment jsdom` pragma scopes jsdom to this file — no global vitest config change required.

```tsx
// @vitest-environment jsdom
import { render, screen } from '@testing-library/react';
import { describe, expect, it } from 'vitest';
import { AppShell } from '../app-shell';

describe('AppShell', () => {
  const renderShell = () =>
    render(
      <AppShell
        header={<div>HEADER_CONTENT</div>}
        aside={<div>ASIDE_CONTENT</div>}
        main={<div>MAIN_CONTENT</div>}
        footer={<div>FOOTER_CONTENT</div>}
      />,
    );

  it('renders the four landmark roles', () => {
    renderShell();
    expect(screen.getByRole('banner')).toBeDefined();
    expect(screen.getByRole('complementary', { name: 'Sidebar' })).toBeDefined();
    expect(screen.getByRole('main')).toBeDefined();
    expect(screen.getByRole('contentinfo')).toBeDefined();
  });

  it('routes each slot into the correct landmark', () => {
    renderShell();
    expect(screen.getByRole('banner').textContent).toBe('HEADER_CONTENT');
    expect(screen.getByRole('complementary').textContent).toBe('ASIDE_CONTENT');
    expect(screen.getByRole('main').textContent).toBe('MAIN_CONTENT');
    expect(screen.getByRole('contentinfo').textContent).toBe('FOOTER_CONTENT');
  });

  it('always renders the unsupported-screen slot (CSS hides it ≥ 768px)', () => {
    const { container } = renderShell();
    const slot = container.querySelector('.app-shell__unsupported');
    expect(slot).not.toBeNull();
    expect(slot?.getAttribute('role')).toBe('alert');
    expect(slot?.textContent).toContain('Desktop only');
  });
});
```
